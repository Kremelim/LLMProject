# Definition of Done

A user story is considered DONE when all relevant items below are completed.

## Code
- [ ] Required function signatures are implemented in app/services.py.
- [ ] Related FastAPI routes are added in app/main.py.
- [ ] Business rules are enforced server-side.
- [ ] No secrets or credentials are hardcoded.
- [ ] Code imports successfully without crashing.

## Database
- [ ] Required PostgreSQL / Supabase tables exist.
- [ ] Data persists correctly.
- [ ] Role and course access data is represented in the database.

## Testing
- [ ] At least one automated test exists under tests/ for the story.
- [ ] Tests pass locally with python -m pytest.
- [ ] Acceptance criteria are verified manually or automatically.

## GitHub
- [ ] Work is committed with traceable message format.
- [ ] Work is pushed to GitHub.
- [ ] Branch / commit / PR names reference story or task IDs when applicable.
- [ ] No .env file or secret is committed.

## ClickUp
- [ ] Related ClickUp task status reflects actual progress.
- [ ] Status transitions are recorded in the Scope Change Log.
- [ ] Board screenshots are saved when required.

## Evidence
- [ ] Required evidence file or screenshot is saved in the evidence folder.
